About this project

Life sciences are experiencing a reproducibility crisis. It has become unexpectedly challenging to even understand how analyses are performed and even more difficult to repeat the same process again.

This site was designed to help scientists share, document, distribute and rerun data analysis scripts. We call these scripts recipes that must adhere to the following requirements:

  • Each recipe must be fully operational.
  • Each recipe must have test data and results associated with it. Hence demonstrating the input requirements as well as the results of the process.
  • Each recipe may be modified, changed and shared across different projects.
Learn more about the way the site works.

Collaborative Analysis

The software that runs this site is called the biostar-engine and it does not favor any particular approach or methodology. Yet it allows users to create full data analysis pipelines that produce the desired results in just a single step.

The software supports running command line oriented tools via a graphical user interfaces generated from each recipe using a JSON specification syntax.

Users may adapt and remix recipes to match their needs, then they may share them back on the site. By supporting these interactions we foster a collaborative environment that facilitates productive interactions and creativity.
